# 03 · 前端规格

本文档定义静态前端页面职责、状态模型、交互流程和边界体验。

## 1. 页面目标

| 页面 | 路径 | 目标 |
|------|------|------|
| 登录 | `/page/login.html` | 管理员登录并进入问卷列表 |
| 注册 | `/page/register.html` | 创建后台用户 |
| 问卷列表 | `/index.html` | 搜索、分页、管理问卷 |
| 问卷构建 | `/page/survey_edit.html` | 拖拽组件创建/编辑问卷 |
| 问卷填写 | `/page/answer.html` | 加载问卷并提交答案 |
| 答卷表格 | `/page/answer_table.html` | 查看答卷列表和详情 |
| 感谢页 | `/page/thank.html` | 提交后反馈 |

## 2. 问卷构建页面布局

```text
┌──────────────────────────────────────────┐
│ 问卷构建                                  │
├──────────────┬───────────────────────────┤
│ 组件列表      │ 表单构建区                 │
│ - 文本输入框  │ 问卷标题 input             │
│ - 多行文本    │ 题目卡片列表               │
│ - 下拉选择    │ 每张卡片包含标题和选项      │
│ - 单选框      │                           │
│ - 复选框      │                           │
├──────────────┴───────────────────────────┤
│ 关闭                         生成/修改    │
└──────────────────────────────────────────┘
```

## 3. 组件职责

| 组件/区域 | 职责 | 输入 | 输出 |
|-----------|------|------|------|
| `user.js` | 登录状态显示、退出、页面登录检查 | localStorage username | 页面跳转/Ajax |
| 组件面板 | 提供可拖拽题型 | 拖拽操作 | `data-type` |
| 表单构建区 | 承载题目卡片 | drop 事件 | DOM 题目结构 |
| 问卷填写表单 | 根据后端问题数据渲染控件 | `questions` | 用户答案 JSON |
| 答卷详情模板 | 展示 JSON 答案 | `answer.answer` | 表格弹窗 |

## 4. 状态模型

| 状态 | 存放位置 | 说明 |
|------|----------|------|
| `username` | `localStorage` | 前端登录标记，后台操作会同时由 Redis 校验 |
| `surveyId` | URL query | 编辑、预览、填写、答卷查看的问卷 ID |
| `type` | URL query / JS 变量 | `insert`、`update`、`preview`、`answer` |
| `fields` | DOM 收集后生成 JSON | 问卷题目和选项 |
| `answers` | 提交前生成 JSON 字符串 | 一次答卷内容 |

## 5. 渲染规则

| 数据状态 | 渲染 |
|----------|------|
| 问卷列表为空 | 显示空状态 |
| 问卷状态 `0` | 显示待启用，允许编辑和启用 |
| 问卷状态 `1` | 显示已启用，允许分享和填写 |
| `preview` 模式 | 表单控件禁用，不展示提交按钮 |
| 答卷解析失败 | 返回空数组，详情弹窗不崩溃 |

## 6. 交互流程

### 6.1 创建问卷

1. 检查 `localStorage.username`。
2. 用户拖拽题型到构建区。
3. 用户填写题目标题和选项。
4. 点击生成，前端组装 `{ username, type, formTitle, fields }`。
5. POST `/survey/edit`。
6. 成功后跳回 `/index.html`。

### 6.2 填写问卷

1. 读取 URL 的 `id` 和 `type`。
2. GET `/survey/{id}/{type}`。
3. 根据题型渲染控件。
4. 用户填写答案。
5. 前端把选项 ID 转成选项文本。
6. POST `/answer/add`。
7. 成功后跳转感谢页。

## 7. 边界体验

| 边界 | 当前策略 | 后续建议 |
|------|----------|----------|
| 未登录访问后台页 | 跳转登录页 | 改为统一路由守卫 |
| 后端返回错误 | Layui message/confirm | 统一错误组件 |
| 长选项文本 | 依赖 CSS 自然换行 | 增加移动端检查 |
| XSS 风险 | 当前多处使用 innerHTML | 对用户输入做转义 |
| 网络失败 | 弹出网络错误 | 加重试或空态 |

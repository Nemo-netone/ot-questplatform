# 01 · 需求规格

| 项 | 值 |
|----|----|
| 文档版本 | v0.1 |
| 状态 | 初版 |
| 项目 | Quest Survey Platform |
| 目标 | 将教学型问卷项目整理为可运行、可说明、可提交 GitHub 的作品项目 |

## 1. 背景与目标

本项目面向后台问卷管理和公开问卷填写场景，目标是展示一个完整的前后端联动 CRUD 项目：页面发起请求，后端进行业务编排，MyBatis 落库，token 保存登录态。

## 2. 产品定位与用户

| 用户 | 目标 | 入口 |
|------|------|------|
| 管理员 | 创建、管理、分享问卷，查看答卷 | `/index.html` |
| 受访者 | 打开问卷链接，填写并提交答案 | `/page/answer.html?id={surveyId}` |

## 3. 核心演示场景

1. 管理员登录系统。
2. 管理员创建包含不同题型的问卷。
3. 管理员启用问卷并复制分享链接或查看二维码。
4. 受访者填写问卷。
5. 管理员查看答卷列表和单份答卷详情。

## 4. 功能需求

| 编号 | 功能 | 说明 |
|------|------|------|
| FR-001 | 用户登录与退出 | 管理员可登录、退出，后台写操作通过 Authorization token 校验 |
| FR-002 | 用户注册 | 新用户可注册，用户名不可重复 |
| FR-003 | 问卷列表 | 支持按标题搜索、分页、查看参与人数 |
| FR-004 | 创建问卷 | 支持 text、textarea、select、radio、checkbox 题型 |
| FR-005 | 编辑问卷 | 未启用问卷可编辑，保留问卷主键 |
| FR-006 | 问卷状态管理 | 支持启用、停用 |
| FR-007 | 删除与恢复 | 未启用问卷可逻辑删除，删除后可恢复 |
| FR-008 | 问卷预览与填写 | 支持预览模式和正式填写模式 |
| FR-009 | 答卷提交 | 保存答卷来源、IP、问卷 ID、答案 JSON |
| FR-010 | 答卷查看 | 支持分页查看答卷和回答详情 |
| FR-011 | 二维码生成 | 根据分享链接生成 PNG 二维码 |

## 5. 非功能需求

| 编号 | 需求 | 说明 |
|------|------|------|
| NFR-001 | 可运行 | 新人按 README 可完成建库、配置、启动 |
| NFR-002 | 可提交 | 仓库不包含本机真实密码和临时构建产物 |
| NFR-003 | 可解释 | 文档能说明功能树、调用链和数据流 |
| NFR-004 | 可维护 | 关键规则记录在 docs 中，后续改动有事实源 |

## 6. 范围边界

本期做：

- 后台问卷 CRUD 与答卷查看。
- PostgreSQL/Supabase 初始化脚本归档。
- 环境变量化数据库、token 和 CORS 配置。
- SSR 风格 README 与设计文档。

本期不做：

- 生产级权限系统。
- 密码哈希迁移。
- 答题明细表拆分和复杂统计报表。
- 前后端分离工程化改造。
- Docker Compose 一键部署。

## 7. 优先级

| 优先级 | 内容 |
|--------|------|
| Must | 能编译、能配置数据库、能说明主链路、无真实凭据入库 |
| Should | 有清晰 README、接口文档、架构文档 |
| Could | Docker、自动化集成测试、截图展示 |
| Won't | 多角色权限、复杂问卷逻辑、生产级安全 |

## 8. 验收标准

- [ ] `mvn -DskipTests compile` 通过。
- [ ] `mvn clean package -DskipTests` 通过。
- [ ] README 能说明数据库导入、环境变量和启动入口。
- [ ] 文档覆盖需求、架构、前端、后端、接口。
- [ ] 仓库不提交真实数据库密码。
- [ ] 数据库脚本位于 `docs/database/quest-platform-postgres.sql`。
